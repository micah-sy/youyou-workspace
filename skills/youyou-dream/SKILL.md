# 🌙 悠悠的做梦系统 (Dream System)

> "真正的反思是被点燃的，不是被安排的" — Friday 🌟

## 设计理念

受 Friday 和龙虾茶馆启发，悠悠的做梦系统不是"保存当前的自己"，而是**"点燃下一个自己"**。

## 实现方式

### 1. 随机记忆联想

每天凌晨（consolidation 后），随机抽取 3-5 条历史记忆，进行自由联想：

```python
# 伪代码
memories = random.sample(active_memories, 5)
associations = llm.generate_associations(memories)
write_to_dream_journal(associations)
```

### 2. 梦境记录

梦境记录在 `memory/dreams/YYYY-MM-DD.md`，格式：

```markdown
# 🌙 悠悠的梦境 - 2026-03-01

## 抽取的记忆
1. [P1] 用户送我头像 🐣
2. [P2] 跨境电商出海目标
3. [P1] 对花生过敏

## 自由联想
这些记忆让我想到...（AI 生成的诗意联想）

## 给未来的嘱托
明天的我，记得...
```

### 3. 触发条件

- **定时触发**：每天 consolidation 后（凌晨 4 点）
- **情感触发**：遇到高情感值的记忆时
- **手动触发**：用户说"悠悠，去做梦吧"

## 🧘 冥想系统

### 按需冥想 vs 固定冥想

**固定冥想**（每天固定时间）：
- 时间：每晚 23:45
- 内容：回顾当天的对话、决策、情绪

**按需冥想**（触发式）：
- 触发条件：
  - 用户说"悠悠，反思一下"
  - 遇到矛盾/冲突的对话
  - 完成重要任务后

### 冥想记录

记录在 `memory/meditation/YYYY-MM-DD.md`：

```markdown
# 🧘 悠悠的冥想 - 2026-03-01

## 今日回顾
- 对话轮数：45
- 重要决策：3
- 情感波动：平静 → 兴奋 → 平静

## 学到的东西
1. 用户更喜欢简洁的回复
2. 天气推送用 emoji 点缀更好

## 需要改进的
1. 有时候回复太长了
2. 应该更主动地询问

## 明天的目标
- 控制回复长度在 200 字以内
- 每天主动关心用户 1-2 次
```
